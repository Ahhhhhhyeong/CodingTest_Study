# 연결 리스트

> **Linked List(연결 리스트)**는 데이터를 “연속된 메모리”에 넣는 대신, **각 요소(노드)가 다음 요소를 가리키며 줄처럼 이어지는 구조**입니다. 배열이 “번호가 붙은 칸”이라면, 연결 리스트는 **“서로 손을 잡고 있는 사람들”**에 더 가깝습니다.

---

## 🧩 기본 구조

각 노드는 보통 두 가지를 가집니다:

1. **데이터**
2. **다음 노드를 가리키는 포인터(참조)**

```text
[데이터 | 다음] → [데이터 | 다음] → [데이터 | 다음] → null
```

* 맨 앞: **Head**
* 맨 끝: **null** (다음이 없음)

---

## 🔗 종류

### 1. Singly Linked List (단일 연결 리스트)

* 한 방향으로만 이동 가능
  → `A → B → C`

### 2. Doubly Linked List (이중 연결 리스트)

* 앞/뒤 모두 이동 가능
  → `A ↔ B ↔ C`

### 3. Circular Linked List (원형 연결 리스트)

* 마지막 노드가 다시 처음을 가리킴
  → `A → B → C → A`

---

## ✅ 장점

### 1. 삽입/삭제가 빠름

* 중간에 끼워 넣기 쉬움 (**O(1)**, 위치만 알면)

```text
A → B → D
→ C를 B 뒤에 삽입
A → B → C → D
```

---

### 2. 크기가 유동적

* 필요할 때마다 노드 추가 가능
* 미리 크기 정할 필요 없음

---

### 3. 메모리 낭비 적음 (유연성 측면)

* 필요한 만큼만 사용

---

## ❌ 단점

### 1. 접근 속도가 느림

* 인덱스로 바로 접근 불가 → **O(n)**

```text
3번째 요소 → 처음부터 하나씩 이동
```

---

### 2. 추가 메모리 필요

* 각 노드마다 “포인터” 저장해야 함

---

### 3. 캐시 효율 낮음

* 메모리에 흩어져 있어서 배열보다 느릴 수 있음

---

## ⚖️ 배열 vs 연결 리스트 핵심 비교

| 항목     | 배열        | 연결 리스트    |
| ------ | --------- | --------- |
| 접근 속도  | 빠름 (O(1)) | 느림 (O(n)) |
| 삽입/삭제  | 느림        | 빠름        |
| 메모리 구조 | 연속        | 비연속       |
| 크기 변경  | 어려움       | 쉬움        |

---

## 🔑 언제 쓰냐?

* **연결 리스트가 유리할 때**

  * 삽입/삭제가 자주 발생
  * 크기가 계속 변함

* **배열이 유리할 때**

  * 조회(읽기)가 많음
  * 인덱스로 빠르게 접근 필요

---

## 한 줄 정리

→ **Linked List = “빠르게 추가/삭제하는 대신, 찾는 건 느린 구조”**
