# 배열

> 배열은 마치 **번호가 매겨진 연속된 칸들이 줄지어 있는 책장**과 같습니다. 각 칸에는 번호(인덱스)가 있어서, "3번 칸에 있는 책(데이터) 꺼내줘!" 하면 바로 찾을 수 있습니다. 컴퓨터 메모리에도 데이터가 실제로 차곡차곡 순서대로 저장됩니다. 

---

배열은 “번호가 붙은 연속된 칸”이라는 구조 덕분에 **속도와 단순함**에서는 강하지만, **유연성**에서는 아쉬움이 있습니다.

---

## ✅ 장점

### 1. 빠른 접근 (핵심 장점)

* 인덱스로 바로 접근 가능 → **O(1)**
* `arr[3]`처럼 즉시 꺼낼 수 있음
  → 책장에서 “3번 칸” 바로 집는 느낌

---

### 2. 메모리 효율 & 캐시 친화적

* 데이터가 **연속된 메모리**에 저장됨
* CPU 캐시 효율이 좋아서 실제 실행 속도도 빠름
  → 특히 C/C++ 같은 언어에서 큰 장점

---

### 3. 구조가 단순함

* 구현과 이해가 쉬움
* 대부분의 자료구조(스택, 큐 등)의 기반

---

### 4. 반복 처리에 최적화

* for문으로 순회하기 좋음

```python id="b1a2c3"
for x in arr:
    print(x)
```

---

## ❌ 단점

### 1. 크기 변경이 어려움 (특히 “진짜 배열”)

* C, Java 배열은 **크기 고정**
* 늘리려면 새 배열 만들고 복사해야 함

---

### 2. 삽입/삭제가 느림

* 중간에 값 넣거나 빼면 **뒤를 전부 밀어야 함 → O(n)**

```text
[1, 2, 3, 4]
→ 2번 위치에 넣기
→ [1, ?, 2, 3, 4] (전부 이동)
```

---

### 3. 메모리 낭비 가능

* 미리 크게 만들면 공간 낭비
* 작게 만들면 다시 생성해야 함

---

### 4. 타입 제약 (언어에 따라)

* C, Java → 한 배열에 한 타입만 가능
* Python/JS는 예외 (혼합 가능하지만 성능 희생)

---

## 🔑 핵심 요약

* **잘하는 것:**
  → 빠른 조회 (읽기 중심)

* **못하는 것:**
  → 삽입/삭제, 크기 변경

---

> 배열은 언어마다 “겉모습”과 “동작 방식”이 조금씩 다르지만, 공통적으로 **인덱스로 빠르게 접근**한다는 핵심은 같습니다. 다만 어떤 언어는 “진짜 배열(고정 크기)”이고, 어떤 언어는 “배열처럼 보이는 유연한 구조”를 기본으로 씁니다.

---

### 🐍 파이썬 (Python)

* **형태:** `list`
* **특징:** 크기 자동 조절, 다양한 타입 섞기 가능

```python
arr = [1, 2, 3]
arr[0]  # 1
```

→ 사실 내부적으로는 “동적 배열”이라서 필요하면 자동으로 확장됩니다.

---

### ☕ 자바 (Java)

* **형태:** `int[]`, `String[]` 같은 **고정 길이 배열**

```java
int[] arr = {1, 2, 3};
arr[0]; // 1
```

* **특징:** 크기 변경 불가
* 대신 가변 구조는 `ArrayList` 사용

---

### ⚡ 자바스크립트 (JavaScript)

* **형태:** `Array`

```javascript
let arr = [1, 2, 3];
arr[0]; // 1
```

* **특징:** 크기 자유롭게 변경, 타입 혼합 가능
  → 사실상 파이썬 list와 비슷하지만 더 느슨함

---

### 🧱 C

* **형태:** 진짜 배열 (메모리에 연속 저장)

```c
int arr[3] = {1, 2, 3};
arr[0]; // 1
```

* **특징:** 크기 고정, 매우 빠름
  → 가장 “교과서적인 배열”

---

### 🧩 C++

* **형태 1:** `int arr[3]` (C 스타일)
* **형태 2:** `vector<int>`

```cpp
vector<int> arr = {1, 2, 3};
```

* **특징:** `vector`는 자동으로 크기 늘어남

---

### 🦀 Rust

* **형태 1:** `[T; N]` (고정 배열)
* **형태 2:** `Vec<T>` (가변 배열)

```rust
let arr = [1, 2, 3];
let v = vec![1, 2, 3];
```

---

### 🐹 Go

* **형태 1:** `[3]int` (배열, 고정)
* **형태 2:** `[]int` (슬라이스, 실전에서 주로 사용)

```go
arr := []int{1, 2, 3}
```

---

### 🍎 Swift

* **형태:** `Array`

```swift
var arr = [1, 2, 3]
```

* **특징:** 타입 안전 + 자동 크기 조절

---

## 🔑 핵심 비교 요약

| ---------- | -------- | ----- | ----- |
| Python     | list     | 가능    | 가능    |
| Java       | 배열       | 불가    | 불가    |
| JavaScript | Array    | 가능    | 가능    |
| C          | 배열       | 불가    | 불가    |
| C++        | vector   | 가능    | 불가    |
| Rust       | Vec      | 가능    | 불가    |
| Go         | slice    | 가능    | 불가    |
